Hello Naomi,

I’m submitting a PixVerse CLI workflow / skill called **PixVerse Shotpack**.

This project is a workflow / skill submission package rather than a standalone app. It turns a structured `brief.md` or `storyboard.yaml` into a reusable shot pack and a Remotion-compatible `manifest.json`. I built it as a practical workflow for agent-driven video generation rather than a one-off prompt demo.

What I made:

- a reusable PixVerse CLI skill spec
- a generic brief and storyboard starter
- a standard T2V shotpack workflow as the main path
- an image-first I2V workflow as an advanced extension for stronger visual consistency
- neutral examples and manifest output examples

How I used PixVerse CLI:

- generated shots from structured inputs instead of ad-hoc prompting
- standardized command usage around `--json`
- handled retries, aspect-ratio passes, output naming, and downstream manifest shaping
- designed the output so it can plug into a Remotion-based editing/rendering workflow

Why this is valuable:

- it shows PixVerse CLI as part of a real production workflow
- it is reusable for creators and AI agents
- it reduces manual setup and makes outputs more predictable
- it connects PixVerse generation with downstream composition cleanly

## 日本語: 公開リポジトリ版

Naomi 様

PixVerse CLI を使った workflow / skill として、**PixVerse Shotpack** を提出します。

このリポジトリは、単体アプリではなく workflow / skill submission パッケージとして、構造化された `brief.md` または `storyboard.yaml` から、再利用可能な shot pack と Remotion 互換の `manifest.json` を生成するためのものです。単発のプロンプト実験ではなく、実際の制作フローや AI Agent で繰り返し使える形を意識して作りました。

今回作ったもの:

- 再利用可能な PixVerse CLI 用 skill 定義
- 汎用の brief / storyboard スターター
- 主導線となる T2V の標準 shotpack workflow
- 世界観統一を重視した image-first I2V の拡張 workflow
- 中立なサンプル入力と manifest 出力例

PixVerse CLI の使い方として重視した点:

- 単発プロンプトではなく、構造化入力からショット生成する形にしたこと
- `--json` を前提にした扱いやすいコマンド運用にしたこと
- リトライ、アスペクト比別生成、出力命名、manifest 整形まで含めて整理したこと
- Remotion ベースの編集 / レンダリング系ワークフローにつなげやすい出力にしたこと

価値があると考えている点:

- PixVerse CLI を実制作フローに組み込む具体例になっていること
- creator と AI Agent の両方で再利用しやすいこと
- 手作業のばらつきを減らし、出力を予測しやすくできること
- 下流の動画合成へ素直につなげられること
