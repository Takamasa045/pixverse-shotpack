# GitHub Release Checklist

Use this before making the repository public.

## Repository Basics

- Repository name is clear
- Description says this is a PixVerse CLI to Remotion workflow / skill
- Visibility is set correctly
- Default branch is clean

## Content Check

- `README.md` is the first thing a reviewer should read
- `README.ja.md` is available for Japanese readers
- `SKILL.md` matches the actual repository contents
- `examples/` is neutral and reusable
- `workflows/` is complete
- `references/` is present

## Remove Noise

- `dist/` is ignored
- `.DS_Store` is ignored
- zip files are ignored
- no secrets or local env files are included
- no client-specific names remain in source docs

## Nice To Have

- Add repository topics such as:
  - `pixverse`
  - `pixverse-cli`
  - `ai-video`
  - `remotion`
  - `workflow`
  - `agent`
- Add one sample MP4 in the GitHub release or external link
- Add one terminal screenshot to the repo description, release, or issue

## What To Send In Email

- Repo link
- Short explanation of:
  - what you made
  - how you used PixVerse CLI
  - why it is valuable
- Optional links:
  - sample MP4
  - terminal screenshot
  - README
  - SKILL
