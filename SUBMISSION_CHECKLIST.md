# Submission Checklist

Use this file before sending the submission.

## Option A: Public Repo

- Public GitHub repo is ready
- `README.md` is readable in under a minute
- `README.ja.md` exists if you want to share a Japanese explanation
- `SKILL.md` is included
- `brief.md` and `storyboard.yaml` are generic starters
- `examples/` contains neutral samples
- `workflows/` contains both main workflows
- `references/` contains manifest and exit-code notes
- `dist/` is not part of the repo submission
- Email includes:
  - what you made
  - how you used PixVerse CLI
  - why it is valuable
  - GitHub repo link

## Option B: Zip Attachment

- Zip contains:
  - `README.md`
  - `README.ja.md`
  - `EMAIL_FINAL.md`
  - `SKILL.md`
  - `brief.md`
  - `storyboard.yaml`
  - `templates/`
  - `examples/`
  - `workflows/`
  - `references/`
- Zip does not contain:
  - `dist/`
  - local caches
  - generated temp files
  - `.DS_Store`
- Email includes:
  - what you made
  - how you used PixVerse CLI
  - why it is valuable
  - note that the repo is attached as zip

## Strong Extras

- 1 sample MP4
- 1 terminal screenshot showing PixVerse CLI usage
- direct link to `README.md`
- direct link to `SKILL.md`

## Final Sanity Check

- The repo looks generic, not tied to a private client project
- The main workflow is obvious from the README
- A reviewer can understand the submission without extra explanation
- The first click gives them either README or SKILL immediately
